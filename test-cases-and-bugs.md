## Часть B. Баги / недоработки

### [Medium] AGG-01: молчаливый сбой обработчиков кнопок
- **TC:** TC-109. **Где:** HomePage/CalendarPage/TasksList/TagPicker (`void onX()`).
- **Суть:** reject async-операции не обрабатывается → кнопка «мертва» без
  обратной связи. Навигационные кнопки при этом работают.
- **Реком.:** try/catch + инлайн-ошибка. **Трудозатраты:** M.

### [Medium] ARCH-02: агрегаторы импортируют соседние модули
- **Где:** home/calendar → tasks/notes. Нарушение «только @core». Без цикла.
- **Реком.:** вынести домен в core или ввести capability-реестр. **M.**

### [Low] CAL-01: нет backfill индекса в календаре/главной (старые волты).
### [Low] SCHED-01: бэклог-задачу нельзя привязать ко дню (после удаления даты из редактора).
### [Low] CAL-02: снимок календаря не обновляется по фокусу.
### [Low] HOME-01: «недавние» по updatedAt вместо lastOpenedAt.

### [Needs-repro] UI-OPEN-01: «половина кнопок не работает»
- В песочнице не воспроизводится (нет браузера). Статически: обработчики на
  месте, файлы целы, типы/линт зелёные.
- **Первое действие:** полный перезапуск `npm run dev` + Ctrl+Shift+R (часто
  лечит после добавления модулей при живом dev-сервере).
- **Если осталось:** F12 → Console → клик «мёртвой» кнопки → прислать ошибку.

## Часть C. Замечания по тестам
- Юнит-тесты модели задач и календаря написаны, но не прогнаны в песочнице
  (нужен `vitest` на машине пользователя). Прогнать `npm test` на Windows.
- Не покрыто автоматизацией: UI-клики (нужен браузер), реальный IndexedDB-сценарий
  привязки/раскраски (только статически).
