# Аудит: новая функциональность (дни/цепочки/теги/главная/календарь)

**Дата:** 2026-06-24 · **Версия:** v0.1.0 (ветка развития)
**Окружение:** песочница Linux, Node 22. `tsc -b --noEmit` — зелёный; ESLint —
чисто на всех новых файлах; чистая логика (даты, сетка месяца, интерполяция
цвета, агрегация дней) — проверена отдельными node-прогонами. Браузерный рантайм
и `vitest` НЕ запускались: `node_modules` собраны под Windows, в Linux-песочнице
нет нативных бинарей esbuild/rollup, а сам каталог недоступен для записи. Поэтому
все находки помечены как **статические** (или подтверждённые точечным прогоном
чистых функций), а UI-дефекты требуют воспроизведения в браузере.

Область аудита: `core/storage/manifest.ts` (v3), `core/storage/collectionRepository.ts`
(`toIndex`), `core/time/dayKey.ts`, `core/tags/*`, `core/shell/Shell.tsx`,
`modules/tasks/*` (модель v2, UI), `modules/home/*`, `modules/calendar/*`.

## Сводка

Находок: Critical 0 · High 0 · Medium 2 · Low 4 · Info 3. Плюс один **открытый
дефект под воспроизведение** («половина кнопок не работает» — в песочнице
повторить нельзя, см. UI-OPEN-01).

Топ к исправлению:
1. AGG-01 (Medium) — молчаливое проглатывание ошибок в async-обработчиках кнопок:
   при сбое операции кнопка «ничего не делает» без обратной связи. Наиболее
   вероятный кандидат на жалобу про «неработающие кнопки».
2. ARCH-02 (Medium) — модули-агрегаторы (home/calendar) импортируют соседние
   модули напрямую, нарушая правило «модуль зависит только от @core».
3. CAL-01/HOME-01 (Low) — нет backfill индекса для старых волтов; «недавние» по
   updatedAt вместо lastOpenedAt.

## Что сделано хорошо

- **Манифест v3 как зашифрованный индекс.** Календарь/главная читают день,
  прогресс и теги из одного расшифрованного конверта на коллекцию — без обхода
  тел. Метаданные остаются зашифрованными на покое (имена тегов в блобах, в
  индексе только непрозрачные id). Модель угроз не ослаблена.
- **Чистые, протестированные модельные слои.** `tasks/model.ts` и
  `calendar/calendarModel.ts` — без побочных эффектов; жёсткая цепочка (блокировка
  шага + сброс последующих) и провал по часам устройства покрыты юнит-тестами.
- **Статус провала — производная, ничего не персистится.** Корректно для
  офлайн-first; нет фонового планировщика, нет рассинхронизации.
- **`exactOptionalPropertyTypes` соблюдён** (день опускается, а не пишется как
  undefined) — индекс не засоряется.

## Находки

### [Medium] AGG-01: молчаливое проглатывание ошибок в обработчиках кнопок
- **Где:** `modules/home/ui/HomePage.tsx` (onQuickAdd), `modules/calendar/ui/CalendarPage.tsx`
  (onQuickAdd), `modules/tasks/ui/TasksList.tsx` (onCreate/onDelete),
  `core/tags/ui/TagPicker.tsx` (onCreate).
- **Класс:** CWE-391 (необработанная ошибка) / UX.
- **Статус:** статический вывод.
- **Суть:** обработчики вызываются как `void onX()`. Если репозиторная операция
  отклонится (волт авто-заблокировался по `visibilitychange`, сбой IndexedDB,
  гонка), reject уходит в `unhandledrejection`, а кнопка визуально «не
  срабатывает» — без ошибки и без действия. При этом кнопки чистой навигации
  (Link/setState) продолжают работать — отсюда эффект «работает половина».
- **Влияние:** при любом сбое записи пользователь не понимает, что произошло;
  диагностика затруднена.
- **Рекомендация:** обернуть async-обработчики в try/catch и показывать ошибку
  (инлайн-баннер/тост); как минимум — `.catch((e)=>setError(...))`. Заодно это
  превратит «мёртвую кнопку» в видимое сообщение.
- **Трудозатраты:** M.

### [Medium] ARCH-02: модули-агрегаторы импортируют соседние модули
- **Где:** `modules/home/ui/HomePage.tsx`, `modules/calendar/ui/CalendarPage.tsx`
  (`import ../../tasks/...`, `../../notes/...`).
- **Класс:** нарушение архитектурного правила (связанность).
- **Статус:** статический вывод.
- **Суть:** заявленное правило — «модули зависят только от @core; ядро не
  импортирует модули». Home/Calendar нарушают его, завязываясь на модель и
  репозиторий tasks/notes. Цикла нет (home/calendar → tasks/notes → core), но
  правило нарушено, и при добавлении ESLint-границы (`no-restricted-imports`) это
  сломает линт.
- **Влияние:** размывается модульность; перенос/удаление модуля tasks потянет
  правки в home/calendar.
- **Рекомендация:** либо вынести доменную модель/репозиторий задач (и заметок) в
  `core/domain` (раз они теперь общие для нескольких модулей), либо ввести
  реестр capability (модуль публикует фабрику репозитория, агрегатор берёт из
  services по ключу). Решение зафиксировать в архитектурных правилах как
  «модули-агрегаторы».
- **Трудозатраты:** M.

### [Low] CAL-01: нет backfill индекса для старых волтов в календаре/главной
- **Где:** `modules/calendar/ui/CalendarPage.tsx`, `modules/home/ui/HomePage.tsx`.
- **Статус:** статический вывод.
- **Суть:** `TasksList` при отсутствии `progress` в индексе делает однократный
  `reindex()`. Календарь/главная — нет: задачи, сохранённые до проектора v3,
  будут без `progress`/`day` в индексе и не попадут в раскраску/«сегодня», пока
  пользователь не зайдёт в список задач.
- **Рекомендация:** вынести общий хелпер «загрузить индекс с backfill» и
  использовать во всех трёх местах.
- **Трудозатраты:** S.

### [Low] SCHED-01: существующую задачу из бэклога нельзя привязать ко дню
- **Где:** `modules/tasks/ui/TaskEditor.tsx` (после удаления выбора даты).
- **Статус:** статический вывод (следствие требования «дата только через
  главную/календарь»).
- **Суть:** создать задачу с днём можно на главной/в календаре; открепить — в
  редакторе. Но привязать ко дню УЖЕ существующую задачу без дня — негде
  (повторное создание теряет шаги/теги).
- **Рекомендация:** в панели дня календаря добавить «прикрепить существующую
  задачу» (выбор из бэклога) или drag-n-drop. 
- **Трудозатраты:** M.

### [Low] CAL-02: снимок индекса не обновляется при возврате из редактора
- **Где:** `CalendarPage.tsx` (entries загружаются на mount).
- **Статус:** статический вывод.
- **Суть:** изменения задачи, сделанные в редакторе, отражаются на календаре
  только при повторном входе (навигация ремонтирует компонент — обычно ок), но
  правки в других вкладках/после фокуса не подтянутся без перезахода.
- **Рекомендация:** перезагружать индекс на фокус окна или по событию навигации.
- **Трудозатраты:** S.

### [Low] HOME-01: «недавние заметки» по updatedAt, а не по открытию
- **Где:** `modules/home/ui/HomePage.tsx`.
- **Статус:** осознанная заглушка (lastOpenedAt — следующий шаг модуля заметок).
- **Рекомендация:** записывать `lastOpenedAt` при открытии заметки и
  проецировать в индекс; на главной сортировать по нему.
- **Трудозатраты:** S.

### [Info] TAG-01: TagPicker расшифровывает все теги на каждый монтаж
- Малые объёмы — приемлемо. При росте — кэшировать карту тегов в контексте.

### [Info] TZ-01: «день» зависит от часов и таймзоны устройства
- Смена таймзоны переинтерпретирует привязку дня и статус провала. Это часть
  принятой модели (доверие к часам устройства) — задокументировать в UI/доках.

### [Info] SEC-NEW: новых утечек на покое нет
- День, прогресс и `tagIds` лежат в зашифрованном манифесте; имена тегов — в
  зашифрованных блобах; в индексе только непрозрачные UUID. XSS-поверхность не
  расширена (нет нового `dangerouslySetInnerHTML`). Инлайн-стили на цветах тегов/
  дней — это значения, не пользовательский HTML.

## Открытый дефект под воспроизведение

### [Needs-repro] UI-OPEN-01: «половина кнопок не работает»
- **Статус:** в песочнице воспроизвести нельзя (нет браузерного рантайма).
- **Проверено статически:** у всех `<button>` есть обработчики; файлы на диске не
  обрезаны; tsc/eslint зелёные; CSS-импорты резолвятся; маршруты/реестр корректны.
- **Гипотезы (по убыванию вероятности):**
  1. **Устаревший dev-сервер Vite.** Новые модули добавлены при работающем
     сервере → `import.meta.glob` + HMR в неконсистентном состоянии. Лечится
     полным перезапуском `npm run dev` и жёстким обновлением (Ctrl+Shift+R).
  2. **AGG-01:** часть кнопок делает операции репозитория; при их сбое — тишина,
     тогда как навигационные кнопки работают (= «половина»).
  3. Авто-лок по `visibilitychange`: переключение вкладки/окна запирает волт;
     после возврата операции падают с LOCKED.
- **Что нужно для точной починки:** открыть DevTools (F12) → Console, кликнуть
  «мёртвую» кнопку и прислать красную ошибку; и какие именно кнопки (меню,
  создание, шаги, дни календаря).
